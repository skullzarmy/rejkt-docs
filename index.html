    REJKT.xyz Feed & Filter Systems - v0.5.6

    REJKT.xyz Feed & Filter Documentation

    This documentation provides a comprehensive overview of the feed and filter systems in REJKT.xyz.

    System Overview

    REJKT.xyz implements multiple feed types with different approaches to displaying NFT content, including Main Feed, Artist Feed, Gallery Feed, and Single Artist Feed. Each feed type has specific characteristics regarding data focus, filtering capabilities, pagination strategy, and persistence mechanisms.

    Changelog

    v0.5.6

    • Fixed issue with FOC (Fully On-Chain) tokens causing excessive network requests
    • Optimized blob URL management system with improved centralized management
    • Added proper cleanup for blob URLs to prevent memory leaks
    • Enhanced documentation for blob URL management components

    v0.5.5

    • Added BlobUrlManager system to prevent memory leaks from blob URLs
    • Fixed issue with excessive network requests when users are idle on feed pages
    • Implemented central tracking and cleanup of blob URLs used for media content
    • Added automatic cleanup routines for orphaned blob URLs
    • Improved memory management for base64 encoded media content

    v0.5.4

    • Added NSFW content filter using _nsfw token metadata attribute
    • Added Flashing Hazards filter using _hazards token metadata attribute
    • Fixed bug where filters would be lost when refreshing feeds
    • Improved filter persistence and application between page reloads
    • Enhanced filter synchronization between UI state and cached content

    v0.5.3

    • Fixed bug where art feed would incorrectly show 'end of feed' after one page
    • Improved feed pagination logic for more reliable content loading
    • Enhanced home page and Tezos information page with additional content
    • Removed filters that were preventing certain FOC art from displaying in feeds
    • Added better error handling and retry mechanisms for feed loading
    • Improved debug logging for feed loading processes
