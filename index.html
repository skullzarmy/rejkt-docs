    REJKT.xyz Architecture - v0.5.6

    REJKT.xyz Architecture Documentation

    REJKT.xyz is a modern platform for discovering and collecting extraordinary digital art on the Tezos blockchain. This documentation provides a comprehensive overview of the application architecture, components, and systems.

    Architecture Overview

    REJKT.xyz is built on a Next.js foundation using React 19 with the App Router, styled with Tailwind CSS, and leverages several specialized systems:

    • Feed Systems: Multiple types of content feeds with different data sources and behaviors
    • Filter Systems: Dynamic filtering capabilities for content discovery
    • Persistence Layer: Client-side state persistence using IndexedDB
    • Blob URL Management: Efficient handling of media assets and base64 data
    • Media Rendering: Support for various media types including images, video, audio, 3D, and text

    System Categories

    Core systems handle the fundamental data flow, business logic, and state management for the application.

    Filter Systems

    The application provides powerful filtering capabilities to help users discover content based on various criteria:

    • Tag-based filtering
    • Artist attributes
    • Token properties
    • Marketplace data
    • Content safety filters

    UI Components

    The interface is built with a component-based architecture following modern React best practices:

    • Feed Components: Display and manage feeds of NFT content
    • Modal Components: Interactive overlays for detailed content viewing
    • Filter Components: UI for applying and managing filters
    • Card Components: Display individual NFT items in feeds
    • Layout Components: Core page structure and navigation

    Utilities

    Utility functions and services that support the application:

    • Blob URL Management: Handles efficient creation and cleanup of blob URLs
    • Data Validation: Ensures data integrity and error handling
    • Media Processing: Transforms and optimizes media content

    State Management

    The application uses a combination of state management approaches:

    • React Context for global state
    • Zustand for specialized state stores
    • Custom hooks for component-specific state

    API Integration

    Integration with blockchain APIs and services:

    • Tezos blockchain API clients
    • NFT marketplace integrations
    • IPFS content resolution
    • Caching strategies

    Media Content Management

    Blob URL Manager

    The application implements a centralized Blob URL management system to handle the creation and cleanup of blob: URLs. This system prevents memory leaks and excessive network requests when users interact with media content, especially base64-encoded NFT assets.

    Key Components

    • BlobUrlManager: Core service that tracks, creates, and cleans up blob URLs
    • useBlobUrl: React hook for easily creating and managing blob URLs in components
    • BlobUrlCleaner: React component for automatic cleanup at regular intervals
    • blobCleanup: Utility functions for manual cleanup operations

    Benefits

    • Prevents memory leaks from orphaned blob URLs
    • Reduces excessive network requests when viewing feed items
    • Improves performance by automatically cleaning up unused resources
    • Provides defensive measures against browser resource exhaustion

    This system is particularly important for feeds with fully on-chain (FOC) NFTs that store media as base64 data directly in the token metadata.

    Changelog

    v0.5.6

    • Fixed issue with FOC (Fully On-Chain) tokens causing excessive network requests
    • Optimized blob URL management system with improved centralized management
    • Added proper cleanup for blob URLs to prevent memory leaks
    • Enhanced documentation for blob URL management components

    v0.5.5

    • Added BlobUrlManager system to prevent memory leaks from blob URLs
    • Fixed issue with excessive network requests when users are idle on feed pages
    • Implemented central tracking and cleanup of blob URLs used for media content
    • Added automatic cleanup routines for orphaned blob URLs
    • Improved memory management for base64 encoded media content

    v0.5.4

    • Added NSFW content filter using _nsfw token metadata attribute
    • Added Flashing Hazards filter using _hazards token metadata attribute
    • Fixed bug where filters would be lost when refreshing feeds
    • Improved filter persistence and application between page reloads
    • Enhanced filter synchronization between UI state and cached content

    v0.5.3

    • Fixed bug where art feed would incorrectly show 'end of feed' after one page
    • Improved feed pagination logic for more reliable content loading
    • Enhanced home page and Tezos information page with additional content
    • Removed filters that were preventing certain FOC art from displaying in feeds
    • Added better error handling and retry mechanisms for feed loading
    • Improved debug logging for feed loading processes
