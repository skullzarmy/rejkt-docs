    Function fetchObjktGraphQL

    • fetchObjktGraphQL<T = any>(
          query: string,
          variables?: Record<string, any>,
          options?: ObjktFetchOptions,
          maxRetries?: number,
      ): Promise<ObjktResponse<T>>

      Fetches data from the OBJKT GraphQL API with resilient error handling

      This is the core function for interacting with the OBJKT marketplace API. It implements several resilience patterns:

      • Automatic retries for failed requests
      • Exponential backoff with jitter to prevent overwhelming the API
      • Consistent error handling and normalization

      The function uses a discriminated union response type that provides type safety when handling both success and error cases.

      Type Parameters

      • T = any

        The expected return type from the GraphQL query

      Parameters

      • query: string

        The GraphQL query string to execute

      • Optionalvariables: Record<string, any>

        Optional variables to include with the GraphQL query

      • Optionaloptions: ObjktFetchOptions

        Additional options for the fetch request

      • maxRetries: number = 15

        Maximum number of retry attempts (default: 15)

      Returns Promise<ObjktResponse<T>>

      A promise resolving to a typed response object

      Example

      // Fetch listings with a price filter
      const response = await fetchObjktGraphQL<ListingQueryResponse>(
        `query GetListings($price: Float) {
          listings(where: { price_lte: $price }) {
            id
            token { name }
            price_xtz
          }
        }`,
        { price: 100 }
      );

      if (response.success) {
        return response.data.listings;
      } else {
        throw new Error(`Failed to fetch listings: ${response.error}`);
      }
