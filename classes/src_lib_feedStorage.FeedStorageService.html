    Class FeedStorageService

    Storage service implementing persistence for all feed types

    This class provides a comprehensive API for storing, retrieving, and managing feed state data across all feed types in the application. It handles data expiration, storage constraints, and cleanup operations automatically.

    The service maintains separate stores for different feed types:

    • Main feed (marketplace listings)
    • Artist feed (artist discovery)
    • Gallery feed (collector galleries)
    • Single artist feed (artist portfolios)

    Each feed type has specialized functions for saving and retrieving state data tailored to its specific requirements and data structures.

    cleanupExpiredStates

    • cleanupExpiredStates(): Promise<void>

      Clean up expired entries to manage storage size

      This maintenance function purges expired cache entries across all feed stores based on the TTL thresholds defined for each feed type. It helps prevent storage bloat and ensures the application stays within browser storage limits.

      The function should be called periodically (e.g., on application startup or after a certain number of user interactions) to maintain storage hygiene.

      Returns Promise<void>

      Promise that resolves when cleanup is complete across all stores

      Example

      // Call during app initialization
      useEffect(() => {
        feedStorage.cleanupExpiredStates();
      }, []);
