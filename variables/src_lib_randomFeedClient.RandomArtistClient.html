    RandomFeedClient for artists

    This object provides specialized functionality for fetching, processing, and managing random artist data for discovery features. It leverages the core artist feed system while adding randomization and filtering logic.

    The client ensures:

    • Artists are unique and not blacklisted
    • Data is appropriately formatted for UI components
    • Cache is used when appropriate to reduce API calls
    • Edge cases are handled gracefully

    Type declaration

    • fetchRandomArtists: function
      • fetchRandomArtists(count?: number): Promise<BaseListing[]>

        Fetches random artists from the OBJKT marketplace

        This method retrieves a diverse selection of random artists for the discovery features. It uses the artist feed infrastructure but applies additional processing to ensure randomness, diversity, and appropriate content:

        1. Fetches a larger pool of artists than requested for better selection
        2. Filters out blacklisted artists and inappropriate content
        3. Ensures uniqueness by wallet address
        4. Randomizes the selection
        5. Applies caching when appropriate

        Parameters

        • count: number = 3

          Number of random artists to return (default: 3)

        Returns Promise<BaseListing[]>

        Promise resolving to an array of unique artist listings

        Example

        const featuredArtists = await RandomArtistClient.fetchRandomArtists(5);
        setFeaturedArtists(featuredArtists);
